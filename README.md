# checkerboard_analyzer
A Python CLI for fitting 4-parameter Hill dose-response curves and calculating Bliss and Loewe drug synergy scores from combination assay data.

## Overview
checkerboard_analyzer reads normalized single drug and combination assay data from an Excel file. Single drug Hill curves are fit for both drugs tested and Bliss and Loewe synergy scores are computed on the combination data. Dose-response plots, synergy heatmaps, and curve fit parameters are saved to the `outputs/` directory.

## Features
* 4-parameter logistic Hill curve fitting via the DoseResponseCurve class, with bounds designed to account for technical noise and prevent Loewe calculation errors
* Calculation of Bliss Independence and Loewe Additivity
* Heatmaps with NaN labels for undefined Loewe cells
* Example datasets and a demo script (demo.py)
* Pytest suite with comparison of Hill fit and synergy matrices to those generated by the synergy Python package

### Usage
```bash
python demo.py --input <input_file.xlsx>
```
Important usage notes:

* Pass the filename only (e.g. ZR751_test.xlsx), not data/ZR751_test.xlsx — files are read from data/

* Outputs go to outputs/<cell_line>/

#### Input data format
Input data should be an Excel workbook with the following structure:

```data``` (sheet 1):

|  Column    |                  Description                  |
| --------   | --------------------------------------------- |
| drug1_conc | Drug 1 concentration (0 for drug-2-only wells)|
| drug2_conc | Drug 2 concentration (0 for drug-1-only wells)|
| response   | Normalized response (e.g. viability 0–1)      |

```assay_info``` (sheet 2):

|  Column    |                Description              |
| --------   | ----------------------------------------|
| drug1_name | Drug 1 name                             |
| drug2_name | Drug 2 name                             |
| cell_line  | Cell line (used for output folder name) |
| conc_units | Concentration units (i.e. μM)           |

For an example, refer to RKO_test.xlsx or ZR751_test.xlsx in the `data` directory.

#### Assumptions
* Responses are already normalized
* Single-drug rows: one concentration column zero, the other > 0
* Combination rows: both concentrations > 0
* Replicates are supported; combination wells are averaged before synergy; single-drug replicates are kept for fit/plot (mean ± SD on dose–response plots)


### Pipeline
1. parse_data() -> single-drug arrays + combination DataFrame + assay metadata
2. DoseResponseCurve.curve_fit() for each drug
3. Synergy.calc_bliss() / calc_loewe()
4. check_fit_quality() in `demo.py`: 

    If Hill slope is deemed unstable (< 0.5 or > 4.0), Loewe is skipped and only Bliss heatmaps are generated

## Models & equations

### Hill equation

$$
E = E_{\min} + \frac{E_{\max} - E_{\min}}{1 + (x / EC_{50})^h}
$$

| Parameter | Meaning |
|-----------|---------|
| **Emax** | High response at low/no drug |
| **Emin** | Low response at high drug |
| **EC50** | Concentration at half-maximal effect |
| **h** | Hill slope (steepness) |

**Mapping to `synergy.single.hill.Hill`:**

| Pipeline | `synergy` Hill |
|----------|----------------|
| Emax | E0 |
| Emin | Emax |
| EC50 | C |
| h | h |

**Fit bounds** (see also [Important caveats](#important-caveatsinterpretation)):

| Parameter | Bounds |
|-----------|--------|
| Emin / Emax | [0, 1.2] |
| EC50 | [0.01 × min tested concentration, 10 × max tested concentration] |
| Hill slope (*h*) | [0.2, 5.0] |

---

### Bliss Independence

$$
\text{score} = E_{\text{obs}} - (E_1 + E_2 - E_1 E_2)
$$

* **E1, E2** — predicted single-agent effects at the combination doses (from fitted Hill curves)
* **E_obs** — observed combination response
* Positive score → less effect than Bliss independence expects (antagonism on this scale)
* Negative score → more effect than expected (synergy on this scale)

**Note:** The `synergy` package uses a different Bliss null model: `E1 × E2 − E_obs`. Scores are **not** directly comparable cell-for-cell.

---

### Loewe Additivity

$$
CI = \frac{d_1}{D_1(E_{\text{obs}})} + \frac{d_2}{D_2(E_{\text{obs}})}
$$

$$
\text{score} = 1 - CI
$$

* **d1, d2** — combination drug concentrations
* **D1(E), D2(E)** — concentrations that would produce combo response **E** on each single-agent curve (Hill inverse)
* **CI = 1** → additive (Loewe additivity)
* **CI < 1** → synergistic (`score > 0`)
* **CI > 1** → antagonistic (`score < 0`)

**Undefined scores:** If combo **E** is outside the fitted range **[Emin, Emax]** for either drug, inversion returns **NaN** and the Loewe score is **NaN** (labeled on heatmaps). On finite cells, pipeline scores match `synergy` `Loewe(mode='ci')` when comparing `1 − CI`.

* Loewe NaN: strong combination kill below single-drug Emin (or above Emax) -> score undefined
* Bliss vs synergy package: different null models; matrices are not directly comparable cell-for-cell
* Loewe vs synergy package: agree on finite cells when Hill fits match; NaN masks align
* Heatmap NaN color: NaN mapped to 0 for coloring -> neutral gray; label displayed as 'NaN'
* Score clipping: values outside [-1, 1] clipped only for visualization
* Hill bounds: Bounds keep fits on a normalized 0–1 (+20% headroom) effect scale, keep EC50 on the assayed dose range, and keep Hill slopes in a range suitable for Loewe synergy analysis; E effect bounds match `synergy` `E_bounds=(0, 1.2)` for comparison with the synergy package. See [Models & equations](#models--equations) for values.

## Limitations & future work
* Currently supports Excel-only input
* Bliss definition differs from synergy package if cross-comparison is needed
* Support of multi-combination and multi-cell line experiments is not currently available, but work is ongoing
* User interface through a web application rather than Python to better support users with limited programming knowledge
